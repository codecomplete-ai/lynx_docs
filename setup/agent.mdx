## Installation

Download the agent binary for Linux or macOS below. 

| **OS**   | **Architecture** | **Download Command**                                                                 |
|----------|------------------|--------------------------------------------------------------------------------------|
| Ubuntu    | x64 (Intel/AMD)  | `wget https://lynx-agent.s3.us-west-2.amazonaws.com/0.0.0/ubuntu/ubuntu_x64_lynx.bin -O lynx.bin` |
| Ubuntu    | arm64 (ARM)      | `wget https://lynx-agent.s3.us-west-2.amazonaws.com/0.0.0/ubuntu/ubuntu_arm64_lynx.bin -O lynx.bin` |


### Running the Agent
After downloading, make the binary executable and run it:
```bash
chmod +x lynx.bin
./lynx.bin
```

## Usage
Run the downloaded program `./lynx.bin` before starting any investigation.

## Authentication
1. Run command 
```bash
./lynx.bin configure
```
2. Generate credential key pair on web by clicking `Generate credential key pair` under `User Settings`.
3. Copy `LYNX_SECRET_ACCESS_ID` and `LYNX_SECRET_ACCESS_KEY` back to lynx. 

### Lynx Config
When you first connect Lynx to your environment, Lynx will automatically generate a config file at `~/.lynx/config` with details about your environment, including OS name and version, CLI tools, and more. 

You can edit this config file directly on your machine to further customize your Lynx experience.

We recommend specifying an absolute path to your codebase in your lynxconfig file or user instructions to help Lynx find your codebase.

Example config:
```yaml
## Lynx Generated Keys
## Do not edit
os_name=Ubuntu
os_version=22.04.5 LTS (Jammy Jellyfish)
aws_cli_version=aws-cli/2.22.12

## User Generated Keys
## Add values here
codebase_dir=/home/ubuntu/myapp
```
